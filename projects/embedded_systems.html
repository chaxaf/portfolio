Embedded Systems Design: Sobel Edge Detection

Hardware/software design project focusing on Sobel edge detection, movement detection, and performance optimization through custom hardware instructions and DMA-based dataflow.

Overview

This project implements a complete image-processing pipeline on an embedded platform. A Sobel filter extracts edges from grayscale images, followed by a movement detection algorithm that compares consecutive frames using configurable thresholds.

Responsibilities

  • Implemented Sobel edge detection and frame-to-frame movement detection.
  • Designed a Sobel accelerator as a custom hardware instruction.
  • Profiled execution cycles to identify performance bottlenecks.
  • Optimized data movement using DMA with SRAM buffer partitioning.

Hardware Acceleration

The Sobel operator was implemented as a custom instruction, using two operands that each encode eight bytes representing neighboring pixels. This reduced instruction count and memory accesses significantly.

Software & Optimization

  • Baseline implementation in C with cycle-accurate profiling.
  • Hardware-accelerated Sobel achieved ~3.25× cycle reduction.
  • DMA-based Sobel optimization reduced CPU cycles by ~15×.
  • Dynamic thresholding added to improve robustness across images.

Technologies

Embedded C, hardware/software co-design, custom instruction logic, DMA-driven dataflow, SRAM buffering, and performance profiling.
